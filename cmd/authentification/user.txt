USER ==> Définit le nom d’utilisateur et d’hôte.

syntax irc : USER <username> <mode> * :<realname>
"USER guest tolmoon tolsun :Ronnie Reagan"

A verifier ==> - Le client n'a pas encore fait de USER

A faire ==> - Sauvegarder les champs
			- Si le client a deja envoyer NICK, il devient registered.

Retour possible :
		
			- (461) ERR_NEEDMOREPARAMS
             "<command> :Not enough parameters"
			  Returned by the server by numerous commands to
              indicate to the client that it didn't supply enough parameters.
			
			- (462) ERR_ALREADYREGISTRED
			  ":You may not reregister"
			  Returned by the server to any link which tries to
              change part of the registered details (such as
              password or user details from second USER message).
